# 🚀 ПРАВИЛЬНЫЙ ЗАПУСК ПРИЛОЖЕНИЯ

## ❌ НЕПРАВИЛЬНО (Вы так делали):
```
Открыли файл в браузере:
file:///E:/Aiprojects/CRM%202026/index.html
↓
Ошибка: CORS и безопасность браузера блокируют запросы
```

## ✅ ПРАВИЛЬНО:

### Шаг 1: Остановите текущий процесс
```bash
# Нажмите Ctrl+C в терминале где работает Node
```

### Шаг 2: Запустите сервер
```bash
cd "e:\Aiprojects\CRM 2026"
node server-standalone.js
```

Вы должны увидеть:
```
🚀 SEO Audit сервер запущен на http://localhost:5000
📍 Откройте браузер и перейдите по адресу выше
```

### Шаг 3: Откройте в браузере
**Нажмите в адресную строку браузера и введите:**
```
http://localhost:5000
```

Или просто кликните сюда: [http://localhost:5000](http://localhost:5000)

---

## 📝 Почему это важно

| Способ | URL | Результат |
|--------|-----|-----------|
| **❌ Файл** | file:///path/to/index.html | CORS ошибки, не работает |
| **✅ Сервер** | http://localhost:5000 | Всё работает ✅ |

Браузер не позволяет JavaScript-у делать HTTP запросы из локальных файлов в целях безопасности. Только через сервер!

1. **Сервер запущен?**
   ```
   Если видите "🚀 SEO Audit сервер запущен" → ✅
   ```

   ```
   ✅ Chrome / Chromium
   ✅ Firefox

   ```

## 🎯 Тестирование

После правильного запуска:

1. Введите URL (например: https://google.com)
2. Нажмите "Запустить анализ"
3. Должен появиться прогресс-бар
4. Через 2-5 сек появится отчет

Если видите ошибку "400 Bad Request", это означает:
- Сервер работает (хорошо!)
- Но есть проблема с форматом запроса (исправлено в обновленном коде)

---

## 💡 Советы

- **Порт 5000 занят?** Отредактируйте server-standalone.js и измените на 5001
- **Браузер кэширует старый файл?** Нажмите Ctrl+Shift+Delete (очистить кэш)
- **Всё ещё ошибка?** Откройте консоль браузера (F12) и посмотрите сообщение об ошибке

---

## 📞 Быстрая помощь

**Ошибка:** EADDRINUSE: address already in use :::5000
**Решение:** Порт занят, используйте другой: `const PORT = 5001;`

**Ошибка:** Cannot find module 'express'
**Решение:** Используйте server-standalone.js (не требует npm)

**Ошибка:** POST 400 Bad Request
**Решение:** Уже исправлено в обновленном коде, просто перезапустите
